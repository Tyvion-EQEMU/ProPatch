# Configuration

ProPatch stores all of its data — settings, database, and logs — in the same folder as `ProPatch.exe`. By default this is `C:\Games\ProPatch\`.

---

## Files

| File | Purpose | Modified by |
|---|---|---|
| `settings.toml` | Shipped defaults | ProPatch (overwritten on self-update) |
| `settings.local.toml` | Your overrides | You / Setup Wizard |
| `gui_settings.json` | GUI state (first-run flag, selected components, log level) | Setup Wizard / GUI |
| `propatch.db` | SQLite database tracking installed versions | ProPatch |
| `propatch.log` | Debug log | ProPatch |

**Your config is safe.** ProPatch only writes `settings.toml` (defaults) during self-update. `settings.local.toml` and `gui_settings.json` are never overwritten — your changes always win.

---

## settings.local.toml

This is the main file you'd edit manually. Any value here overrides the shipped defaults in `settings.toml`.

### GitHub Token

Raises the GitHub API rate limit from 60 to 5,000 requests per hour.  Was useful when developing this utility, but you may never need or use this.

```toml
github_token = "ghp_your_token_here"
```

To create a token: [github.com/settings/tokens](https://github.com/settings/tokens) → Generate new token (classic) → no scopes needed.

### Paths

```toml
[paths]
mq_rekkas = "C:\\Games\\MQ-Profusion"
eq_dirs   = ["C:\\Games\\EverQuest"]
```

Multiple EQ directories (e.g., separate installs for different characters):

```toml
[paths]
mq_rekkas     = "C:\\Games\\MQ-Profusion"
eq_dirs       = ["C:\\Games\\EQ_Tanks", "C:\\Games\\EQ_Boxes"]
eq_dir_names  = ["Tanks", "Boxes"]
```

`eq_dir_names` is optional — friendly labels used in log output.

### Disabling Components

```toml
[components]
proloot  = false    # skip ProLoot entirely
rgmercs  = false    # skip RGMercs entirely
```

Any component set to `false` is skipped during scans and updates. You can also just uncheck the checkbox in the GUI — the effect is the same.

---

## settings.toml

Shipped with ProPatch and contains the baseline defaults. **Do not edit this file** — it gets overwritten when ProPatch self-updates. Put all your changes in `settings.local.toml` instead.

---

## gui_settings.json

Managed by the GUI and Setup Wizard. You generally don't need to edit this by hand, but in a pinch it's plain JSON:

> **Note:** MQ path and EQ directories are stored in `settings.local.toml` under `[paths]`, not here. The GUI reads them directly from that file so both the GUI and CLI always share the same source of truth.

```json
{
  "first_run_complete": true,
  "propatch_install_path": "C:\\Games\\ProPatch",
  "install_mq": true,
  "selected_components": ["propatch", "rekkas_mq", "mq2rwarp", "rgmercs", "proloot",
                          "spells_us", "dbstr_us", "skillcaps", "basedata", "dinput8"],
  "custom_components": [],
  "log_level": "INFO"
}
```

---

## Resetting to Defaults

To fully reset ProPatch (wipe all settings and start the Setup Wizard fresh), delete everything in the ProPatch folder **except** `ProPatch.exe` and relaunch.

To reset just the GUI state, delete `gui_settings.json`. The Setup Wizard will run on next launch.
