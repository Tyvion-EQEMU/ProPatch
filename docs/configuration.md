# Configuration

ProFetch stores all of its data — settings, database, and logs — in the same folder as `profetch.exe`. By default this is `C:\Games\ProFetch\`.

---

## Files

| File | Purpose | Modified by |
|---|---|---|
| `settings.toml` | Shipped defaults | ProFetch (overwritten on self-update) |
| `settings.local.toml` | Your overrides | You / Setup Wizard |
| `gui_settings.json` | GUI state (paths, selected components, EQ instances) | Setup Wizard / GUI |
| `profetch.db` | SQLite database tracking installed versions | ProFetch |
| `profetch.log` | Debug log | ProFetch |

**Your config is safe.** ProFetch only writes `settings.toml` (defaults) during self-update. `settings.local.toml` and `gui_settings.json` are never overwritten — your changes always win.

---

## settings.local.toml

This is the main file you'd edit manually. Any value here overrides the shipped defaults in `settings.toml`.

### GitHub Token

Raises the GitHub API rate limit from 60 to 5,000 requests per hour.  Was useful when developing this utility, but you may never need or use this.

```toml
github_token = "ghp_your_token_here"
```

To create a token: [github.com/settings/tokens](https://github.com/settings/tokens) → Generate new token (classic) → no scopes needed.

### Paths

```toml
[paths]
mq_rekkas = "C:\\Games\\MQ-Profusion"
eq_dirs   = ["C:\\Games\\EverQuest"]
```

Multiple EQ directories (e.g., separate installs for different characters):

```toml
[paths]
mq_rekkas     = "C:\\Games\\MQ-Profusion"
eq_dirs       = ["C:\\Games\\EQ_Tanks", "C:\\Games\\EQ_Boxes"]
eq_dir_names  = ["Tanks", "Boxes"]
```

`eq_dir_names` is optional — friendly labels used in log output.

### Disabling Components

```toml
[components]
proloot  = false    # skip ProLoot entirely
rgmercs  = false    # skip RGMercs entirely
```

Any component set to `false` is skipped during scans and updates. You can also just uncheck the checkbox in the GUI — the effect is the same.

---

## settings.toml

Shipped with ProFetch and contains the baseline defaults. **Do not edit this file** — it gets overwritten when ProFetch self-updates. Put all your changes in `settings.local.toml` instead.

---

## gui_settings.json

Managed by the GUI and Setup Wizard. You generally don't need to edit this by hand, but in a pinch it's plain JSON:

```json
{
  "first_run_complete": true,
  "profetch_install_path": "C:\\Games\\ProFetch",
  "install_mq": true,
  "install_path": "C:\\Games\\MQ-Profusion",
  "eq_instances": [
    { "path": "C:\\Games\\EverQuest", "name": "Main" }
  ],
  "selected_components": ["profetch", "rekkas_mq", "mq2rwarp", "rgmercs", "proloot",
                          "spells_us", "dbstr_us", "skillcaps", "basedata", "dinput8"]
}
```

---

## Resetting to Defaults

To fully reset ProFetch (wipe all settings and start the Setup Wizard fresh), delete everything in the ProFetch folder **except** `profetch.exe` and relaunch.

To reset just the GUI state, delete `gui_settings.json`. The Setup Wizard will run on next launch.
